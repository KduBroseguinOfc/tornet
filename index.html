Quick Start

Zero to rotating in 30 seconds

TorNet wraps Tor's control protocol into a clean Python API. No config files. No manual setup.

tornet — zsh 
# CLI usage — change IP every 60 seconds, 10 times
$ tornet --interval 60 --count 10

[*] Initializing Tor environment...
[+] Tor is running on port 9050
[+] Current IP: 185.220.101.47
[~] Rotating identity...
[+] New IP:     104.244.73.80
[~] Rotating identity...
[+] New IP:     199.87.154.255
...

# Or use it as a Python library
$ python3
>>> from tornet import initialize_environment, ma_ip, change_ip
>>> initialize_environment()
True
>>> ma_ip()
'185.220.101.47'
>>> change_ip()
'104.244.73.80'
>>> █
Features

What TorNet does

A minimal, focused library — no bloat, no unnecessary dependencies.

Automated IP Rotation
Cycle through Tor exit nodes at a configurable interval. Set count to 0 for indefinite rotation.
Python API
Import and use directly in your scripts. change_ip(), ma_ip(), and more.
CLI Interface
Run from any terminal with tornet commands. No Python script required.
Docker Support
Official Dockerfile and docker-compose for isolated, reproducible deployments.
Auto-Fix
Missing dependencies? Run tornet --auto-fix to resolve common setup issues automatically.
Browser Proxy
Configure Firefox to route traffic through TorNet's SOCKS5 proxy on 127.0.0.1:9050.

Installation

TorNet requires Python 3.7+ and Tor installed on your system.

Install via pip

bash 
pip install python-tornet

Install via AUR (Arch Linux)

bash 
yay -S python-tornet

Install Tor

TorNet requires the Tor daemon to be available on your system:

bash 
# Debian/Ubuntu
sudo apt install tor

# Fedora/RHEL
sudo dnf install tor

# Arch Linux
sudo pacman -S tor

# macOS
brew install tor

Verify installation

bash
$ tornet --ip
185.220.101.47
NOTE — If you encounter dependency issues, run tornet --auto-fix to resolve them automatically.

CLI Reference

TorNet exposes a full command-line interface after installation.

Commands

Flag Type Description
--ipflagDisplay your current Tor exit node IP address
--interval <sec>integerSeconds between each IP rotation
--count <n>integerNumber of rotations. 0 = run indefinitely
--stopflagStop all running TorNet processes
--auto-fixflagAutomatically install missing dependencies

Examples

bash
# Show current IP
$ tornet --ip

# Rotate every 60s, exactly 10 times
$ tornet --interval 60 --count 10

# Rotate every 30s indefinitely
$ tornet --interval 30 --count 0

# Stop all TorNet processes
$ tornet --stop

# Fix broken dependencies
$ tornet --auto-fix

Verify your Tor circuit

bash
curl --socks5 127.0.0.1:9050 https://check.torproject.org/api/ip

Python API

Import TorNet directly into your Python scripts for programmatic control.

Import

python
from tornet import ma_ip, change_ip, initialize_environment, change_ip_repeatedly

Functions

initialize_environment() required

Starts the Tor daemon and sets up the SOCKS proxy. Must be called before any other function.

python
initialize_environment()  # Returns True on success

ma_ip() → str

Returns your current IP address as seen from the Tor exit node.

python
ip = ma_ip()
print(ip)  # '185.220.101.47'

change_ip() → str

Requests a new Tor identity and returns the new exit node IP.

python
new_ip = change_ip()
print(new_ip)  # '104.244.73.80'

change_ip_repeatedly(interval, count)

Rotates IP on a schedule. Pass count=0 to run indefinitely.

ParameterTypeDescription
intervalintSeconds between rotations
countintNumber of rotations. 0 = indefinite
python
change_ip_repeatedly(60, 10)   # Every 60s, 10 times
change_ip_repeatedly(30, 0)    # Every 30s, forever

Full example

python
from tornet import ma_ip, change_ip, initialize_environment

# Initialize Tor
initialize_environment()

# Print current IP
print(f"Current IP: {ma_ip()}")

# Rotate and print new IP
print(f"New IP:     {change_ip()}")

Docker

TorNet ships with an official Dockerfile and docker-compose configuration for isolated deployments.

Using docker-compose

bash
# Clone the repo
git clone https://github.com/ayadseghairi/tornet.git
cd tornet

# Build and run with docker-compose
docker compose up --build

Using Docker directly

bash
# Build the image
docker build -t tornet .

# Run with interval and count
docker run --rm tornet --interval 60 --count 5
NOTE — Docker deployments expose the SOCKS5 proxy on port 9050. Map it with -p 9050:9050 if you need host access.

Browser Configuration

Route your browser traffic through TorNet's SOCKS5 proxy.

Firefox

With TorNet running, configure Firefox to use it as a SOCKS5 proxy:

  1. Open Preferences → Network Settings
  2. Select Manual proxy configuration
  3. Set SOCKS Host to 127.0.0.1
  4. Set Port to 9050
  5. Select SOCKS v5
  6. Check Proxy DNS when using SOCKS v5
  7. Click OK

Verify in browser

Visit check.torproject.org to confirm your traffic is going through Tor.

NOTE — DNS leaks are prevented when "Proxy DNS" is enabled. Always verify with the Tor Project's checker.

How Tor Exit Nodes Work — and Why IP Rotation Matters

When you connect to the Tor network, your traffic passes through three relays before reaching the destination: a guard node (entry), a middle relay, and an exit node. The exit node is what the destination server sees as your IP address.

Each time you request a new Tor identity — which TorNet does programmatically via the control protocol — Tor builds a new circuit with a different exit node. From the perspective of any server you're connecting to, you appear to be a different machine from a different country.

Why this matters for OSINT

Open-source intelligence gathering often involves repeated, automated requests to public web services. Rate limiting and IP-based blocking are the first line of defense these services deploy. By rotating through Tor exit nodes, you can sustain data collection workflows that would otherwise be throttled or banned.

TorNet automates this entirely: call change_ip() or let change_ip_repeatedly() handle the timing, and your scripts get a fresh identity on demand.

Responsible use

IP rotation through Tor is a legitimate tool for privacy, security research, and OSINT — but it carries ethical weight. Never use TorNet to bypass protections in ways that harm services or violate terms of service. The Tor network is a shared resource maintained by volunteers.

Building TorNet: From Script to PyPI Package

TorNet started as a small utility script — a few lines of Python that called Tor's control port to request a new circuit. No CLI, no packaging, just something that worked in a terminal session.

The turning point was realizing that the pattern was repeatable. Every time I needed to automate requests against a public API, I copied the same block of code. That repetition is the clearest signal that something should become a library.

The packaging decision

Getting a package onto PyPI is straightforward, but the naming matters. python-tornet was chosen to be unambiguous in a pip install context, while the import name stays clean at from tornet import ....

The AUR package came later, after Arch users started filing issues about installation paths. Maintaining a PKGBUILD turned out to be less work than answering the same setup question repeatedly.

What 260 stars taught me

Most of the GitHub traffic came from two places: people searching for "Tor IP rotation Python" and mentions in OSINT resource lists. Neither was planned. The lesson: solve a specific problem well, name it clearly, and put it where people search.

OSINT Automation with Python and Tor

A practical OSINT workflow combines passive reconnaissance tools with anonymized request infrastructure. TorNet fits into this stack as the transport layer — handling IP rotation so your tools can focus on what they're good at.

The basic stack

Three tools cover most passive recon needs: subfinder for subdomain enumeration, Amass for deeper DNS graph mapping, and WhatWeb for technology fingerprinting. All three can be scripted in Python and wrapped with TorNet rotation.

Wiring it together

The pattern is simple: initialize TorNet, run a batch of requests, rotate identity, repeat. For active probing (HTTP requests, API calls), route everything through the SOCKS5 proxy at 127.0.0.1:9050 using the requests library's proxies parameter.

python
import requests
from tornet import initialize_environment, change_ip

initialize_environment()

proxies = {'http': 'socks5h://127.0.0.1:9050', 'https': 'socks5h://127.0.0.1:9050'}

for target in targets:
    r = requests.get(target, proxies=proxies)
    # process r.text ...
    change_ip()  # fresh identity per target

Staying ethical

OSINT is legal when limited to publicly available information and used for legitimate purposes — security research, threat intelligence, bug bounty reconnaissance within defined scope. Always verify you have authorization before active probing, and respect robots.txt.
